CRA Compliance Self Assessment

Beoordeel uw CRA-compliance in 10 vragen. Ontdek waar uw embedded Linux systeem kwetsbaar is en welke stappen u moet nemen voor 11 september 2026.

Vraag 1/10

Genereert uw Yocto/Buildroot build automatisch een SBOM?

WETTELIJK VERPLICHT: Art. 14 CRA vereist een volledige SBOM van alle packages, kernel modules en firmware components in uw embedded Linux systeem. Boetes tot €15M bij non-compliance.

Vraag 2/10

Scant u Linux kernel + userspace packages op CVE's?

24-UUR MELDPLICHT: Art. 14 CRA vereist rapportage van actief geëxploiteerde kwetsbaarheden aan CSIRT binnen 24 uur. Embedded Linux heeft vaak 100+ kwetsbare packages.

Vraag 3/10

Heeft u U-Boot Verified Boot + dm-verity?

ANNEX I VERPLICHTING: Embedded Linux moet bootloader signature verification (U-Boot FIT images) + runtime root filesystem integrity (dm-verity) hebben. Zonder dit: geen CE-markering mogelijk.

I
Vraag 4/10

Gebruikt u SWUpdate/Mender/RAUC met signature verification?

ANNEX I §2: OTA-updates voor embedded Linux moeten cryptografisch gesigned zijn (RSA/ECDSA), rollback-protected (A/B partities) en gefaalde updates moeten automatic recovery hebben. Anders: productaansprakelijkheid.

Vraag 5/10

Gebruikt u hardware crypto (TPM/TEE) voor key storage?

ANNEX I §3: Embedded Linux moet cryptografische keys opslaan in hardware (TPM 2.0, ARM TrustZone TEE, secure elements). Software-only key storage in rootfs is non-compliant en extractable.

14
Vraag 6/10

Heeft u een TCF voor uw embedded Linux BSP?

ART. 13 BEWAARPLICHT: Embedded Linux TCF moet bevatten: Yocto layer documentatie, kernel config rationale, SBOM, penetration test reports, en threat model. Bewaarplicht 10 jaar - markttoezicht kan dit opvragen.

Vraag 7/10

Screent u alle Yocto layers & BSP vendors op CVE's?

SUPPLY CHAIN AANSPRAKELIJKHEID: U bent verantwoordelijk voor kwetsbaarheden in NXP/Qualcomm/TI BSPs, meta-openembedded layers, en closed-source firmware blobs. 1 kwetsbare binary = volledige product recall.

13
Vraag 8/10

Kan uw embedded device patches ontvangen in het veld?

ART. 14 INCIDENT RESPONSE: Embedded Linux devices moeten een coordinated vulnerability disclosure proces hebben + ability to patch deployed devices. Devices zonder update-mechanisme zijn non-compliant.

Vraag 9/10

Heeft u een LTS kernel met 5+ jaar support?

ART. 11 MINIMALE SUPPORT: Embedded Linux moet LTS kernel gebruiken (bijv. 6.6 LTS tot 2029). Vendor BSPs met 2-jaar support zijn non-compliant. U moet kernel patches 5+ jaar leveren of civielrechtelijk aansprakelijk.

11
Vraag 10/10

Heeft u debug interfaces (UART/JTAG) disabled in productie?

ANNEX I ACCESS CONTROL: Embedded Linux moet debug interfaces (UART root shell, JTAG, USB gadget mode) disablen in productie builds. Open debug ports = directe root access = critical vulnerability.
